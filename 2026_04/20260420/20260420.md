# static & media

# Static files
- 서버 측에서 변경되지 않고 고정적으로 제공되는 파일
- 항상 같은 이미지나 logo를 제공하는 파일
- 웹 서버가 제공하는 가장 기본적인 자원
- 기본 경로 : app폴더/static
  
### 처리 과정 요약
1. 사용자가 주소를 입력하여 이미지 요청
2. 웹서버는 URL을 확인하고 미리 약속된 폴더에서 파일을 찾음.
3. 찾은 이미지 파일을 http 응답에 담아 사용자에게 전송
4. 브라우저가 응답 받은 이미지 파일을 화면에 보여줌.


### 기본 경로 CSS 스타일 제공하기
- articles/static/stylesheets/ 경로에 CSS 파일 배치하기
- 메인 페이지에서 CSS 파일 불러오기 
```html
<!-- articles/index.html -->
{% load static %}

<link rel='stylesheet' href="{% static 'stulesheets/style.css' %}">



<img src="{% static "images/sample-1.png" %}" alt="sample image"> : sample-1.png를 static/images/sample-1.png에서 불러오겠다.

```
```python 
STATICFILES_DIRS = [
    BASE_DIR / 'static', # 공통으로 사용하는 static file 경로
]
```
- loda static : static 태그 사용 위해 사용하겠다고 알려주는 선언문
- CSS가 적용된 것을 확인, static tag가 어떤 주소를 만들어 냈는지 확인
- STATICFILES_DIRS : static file을 base.html에 등록하기 위해서 정해야 할 파
### settings.py
- STATIC_URL = 'static/' : static입력을 받으면 그에 대한 STATIC을 연결시킴.

# Media files
- 사용자가 웹 사이트를 통해 직접 업로드하는 파일
- ImageField() : 이미지 파일을 업로드하기 위해 사용되는 django 모델 필드
```python
class Article(modes.Model):
    image = model.ImageField(upload_to='images/') # DB에는 images/sample.png와 같은 같은 경로 문자열 저장, 이미지는 MEDIA_ROOT 경로에 저장
```
# MEDIA_ROOT와 MEDIA_URL 설정
- MEDIA_ROOT = 파일을 저장하고 관리하기 위한 서버의 실제 경로, 
- MEDIA_URL = 그 파일을 웹에서 보여주기 위한 가상의 주소
```python
MEDIA_ROOT = BASE_DIR / 'media'
MEDIA_URL = '/media/'
```
```python
from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    # ... the rest of your URLconf goes here ...
] + static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
```
- settings.py의 Static files의 사이트에서 위의 코드가 나옴(굳이 외울 필요 x)

### 날짜를 이용한 경로
- starttime()의 형식 코드
- 파일이 업로드된 날짜를 기준으로 폴더를 자동으로 생성할 수 있다.
1. %Y: 4자리 연도, %m: 2자리 월, %d: 2자리 일
```python
# 예시
class Photo(models.Model):
    # 2100년 1월 1일에 업로드하면 '2100/01/01/' 폴더에 저장됨.
    image = models.ImageField(blank=True, upload_to='%Y/%m/%d/')
```
