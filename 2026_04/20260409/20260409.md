# Django : Intro & Design Pattern(2026.04.09)

# 웹 작동 방식
- client <-> server
- client : 서비스를 요청(request)하는 주제 / server : 클라이언트의 요청에 응답(response)하는 주체
- 예시 : 우리가 웹 페이지를 보게 되는 과정
1. 클라리언트가 url을 입력한 후 enter
2. 인터넷에 연결된 구글 컴퓨터(서버)에게 메인 홈페이지.html을 달라고 요청
3. 구글 컴퓨터는 데이터베이스에서 메인 홈페이지.html 파일 찾아 응답.
4. 웹 브라우저는 사람이 이를 볼 수 있도록 해석해주고 사용자는 구글의 메인 페이지를 보게 된다.

# frontend vs backend
- 프론트엔드 : 사용자 인터페이스(UI)를 구성하고, 사용자가 애플리케이션과 상호작용할 수 있도록 한다.
- HTML, CSS, JavaScript, 프론트엔드 프레임워크 등

- 백엔드 : 서버 측에서 동작하며, 클라이언트의 요청에 대한 처리와 데이터베이스와의 상호작용 등을 담당
- 사용자가 봐야하는 데이터를 전달함.
- 서버 언어(Python, Java) 및 백엔드 프레임워크, 데이터베이스, API, 보안 등

# Web Framework
- 웹 애플리케이션을 빠르게 개발하도록 도와주는 도구
- 로그인/로그아웃, 회원관리, 데이터베이스, 보안 등

# Django
- Python 기반의 대표적인 웹 프레임워크
- 목표 : 클라이언트, 서버 구조의 서버를 구현하는 것!
- 다양성, 확장성, 보안, 커뮤니티 지원

# 가상 환경
- 하나의 컴퓨터 안에서 또 다른 독립된 파이썬 환경
- 여러 개의 프로젝트를 진행할 경우 각각의 가상 환경에서 진행(버전 충돌 방지)
- 1개의 프로젝트 당 1개의 가상환경
- 생성방법 : "$ python -m venv(명령어) venv(가상환경 이름, 변경 가능)" -> 현재 디렉토리 안에 venv라는 폴더가 생성됨.
- 활성화 : $ source venv/Scripts(폴더명)/activate -> 프롬프트 안에 (venv)와 같이 표시된다면 성공한 것
- 예시 : PJT A를 활성화를 하겠다? -> 1. cd PJTA -> 2. source venv/Scripts/activate
- 가상 환경 종료 : $ deactivate -> 다시 Python Global 환경으로 돌아온다.
- $ pip list : 현재 가상 환경에 설치된 라이브러리 목록을 확인하는 명령어

### 의존성 패키지
- 프로젝트가 의존하는 개별 라이브러리들을 가리키는 말
- 프로젝트가 실행되기 위해 필요한 각각의 패키지
- 의존성 기록(내보내기) : $ pip freeze > requirements.txt -> 현재 가상환경에 설치된 모든 패키지와 버전을 파일로 저장
- pip freeze : 가상 환경에 설치된 모든 패키지를 버전과 함께 특정한 방식으로 출력하는 명령어
- 의존성 설치(가져오기) : $ pip install -r requirements.txt -> requeirements.txt에 명시된 라이브러리를 한 번에 설치(다른 사람, 혹은 내가 requirements.txt 파일을 받았을 때 그 안에 있는 파일을 설치)

- 2명의 개발자가 하나의 프로젝트를 같이 개발할 때 Github에 push를 진행하면 clone받고 실행이 안됨(패키지 실행 X) -> 그러나 개발한 사람이 의존성 기록을 진행했다면 clone할 때 패키지 목록이 공유되어 실행 가능함.

### 가상환경 주의사항
1. 가상환경은 파이썬 환경을 온오프하는 개념
2. 프로젝트마다 별도의 가상 환경 사용
3. 가상 환경 폴더 venv는 관련된 프로젝트와 동일한 경로에 위치시킴
4. 폴더 venv는 .gitignore파일에 작성되어 원격 저장소에 공유하지 않음.(대신 requirements.txt를 공유함.) -> 각자의 가상환경 구성

# Django 프로젝트 시작(예시)
1. python -m venv venv (가상 환경 생성 : 현재 디렉토리 안에 venv라는 폴더가 생성됨.)
2. source venv/Scripts/activate
3. pip install django (django 설치)
4. django-admin startproject firstpjt . (프로젝트 폴더 생성), ('.'은 현재 위치라는 의미, firstpjt는 프로젝트 이름)
5. python manage.py runserver(서버 실행)
Ctrl + c
6. python manage.py startapp articles(articles라는 폴더와 내부에 여러 파일이 생성됨.)
(중요!) 반드시 앱을 생성한 후에 등록해야함.


# Django 패턴
- 디자인 패턴 : 소프트웨어 설계에서 반복적으로 발생하는 문제에 대한, 검증되고 재사용 가능한 일반적인 해결책(MVC(Model(데이터 및 비즈니스 로직 처리), View(사용자에게 보여지는 화면), Controller(M, V 제어)) 대표적)
- MTV 디자인 패턴 : Django에서 애플리케이션을 구조화하는 디자인 패턴(View를 Template, Controller를 View로)
- 프로젝트와 앱 : 프로젝트 내부의 여러 기능을 묶은 앱(독립적으로 작동하는 기능 단위 모듈)으로 구성

### 프로젝트 구조
- settings.py : 프로젝트의 모든 설정을 관리
- urls.py : 요청 들어오는 URL에 따라 이에 해당하는 적절한 views를 연결

### 앱 구조(수정할 것)
- admin.py : 관리자용 페이지 설정
- models.py : DB와 관련된 모델 정의, MTV의 M
- views.py : HTTP 요청 처리하고 해당 요청에 대한 응답을 반환, MTV의 V

### Django와 요청 & 응답
- 사용자가 요청한 명령이 urls.py에 존재하는지 여부 확인 -> app에 존재하는 views.py에 들어가 model, template와 교류하고 응답을 진행
