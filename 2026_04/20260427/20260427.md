# DB

# 데이터베이스
- 데이터베이스 : 체계적으로 정리된 데이터의 모음
- 데이터 : 저장이나 처리를 위해 변환된 정보
- 관계형 데이터베이스 : 데이터 간에 관계가 있는 데이터 항목의 모음
- 관계형 DB의 예시 : id에 고객 ID를 부여하여 고객 테이블 정보 파악

# 관계형 데이터베이스 관련 키워드
1. Table(aka Realtion) : 데이터를 기록하는 곳
2. Field : 각 필드에 고유한 데이터 형식 저장
3. Record : 각 레코드에 구체적인 데이터 값 저장
4. Database : 테이블의 집합
5. Primary Key(기본 키) : 각 레코드의 고유한 값, 레코드의 식별자로 활용
6. Foreign Key(외래 키) : 테이블의 필드 중 다른 테이블의 레코드를 식별할 수 있는 키 -> 관계를 만드는 데 사용

# DBMS
- 데이터 저장 및 관리를 용이하게 하는 시스템
- RDBMS : 관계형 데이터베이스 관리 소프트웨어 프로그램
- RDBMS 서비스 종류 : SQLite, MySQL, PostgreSQL -> 모두 SQL 기반으로 작동
- SQLite : 경량의 오픈 소스 DB 관리 시스템

# SQL
- Structure Query Language : 테이블의 형태로 구조화 된 관계형 데이터베이스에게 요청을 질의(요청)
- Syntax : SELECT column_name FROM table_name; (SELECT statements : SELECT, FROM 2개의 키워드로 구성)
1. SQL 키워드는 대소문자를 구분하지 않음
2. 각 SQL statements의 끝에는 세미콜론(;)필요 !
- SQL statement : SQL을 구성하는 가장 기본적인 코드 블록
- 4가지 유형
1. DDL : 데이터의 기본 구조 및 형식 변경(CREATE, DROP, ALTER)
2. DQL : 데이터 검색(SELECT)
3. DML : 데이터 조작(INSERT, UPDATE, DELETE)
4. DCL(안 다룸) : 사용자 권한 제어(COMMIT, ROLLBACK, GRANT, REVOKE)

# Querying Data
- Database 연결 : Database Client 사용

### SELECT syntax
- SELECT 키워드 이후 데이터를 선택하려는 필드를 하나 이상 지정
- FROM 키워드 이후 데이터를 선택하려는 필드 지정
```sql
SELECT
  select_list
FROM
table_name;
```
- 활용 1: 테이블 employees에서 LastName 필드의 모든 데이터 조회
```sql
SELECT LastName, Firstname FROM employees;
```

- ORDER BY syntax
```sql
SELECT
  FirstName
FROM 
 employees
ORDER BY
ReportsTo;
```
- DISTINCT syntax : 조회 결과에서 중복된 레코드를 제거

### wildcard characters
- '%' : 0개 이상의 문자열과 일치 하는지 확인
- '_' : 단일 문자와 일치하는지 확인
