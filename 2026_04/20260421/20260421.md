# Authentication System(인증 시스템)

# HTTP
- HTML문서와 같은 리소스들을 가져올 수 있도록 해주는 규약
- 특징
1. 비연결 지향: 서버는 요청에 대한 응답을 보낸 후 응답을 끊음. **클라이언트는 서버와 연결되어 있는 상태가 아니다!**
2. 무상태 : 연결을 끊는 순간 클라이언트와 서버 간의 통신이 끝난다. / 상태 정보가 유지되지 않는다. -> 로그인 상태를 유지할 수 없다.

# 쿠키
- 서버가 사용자의 웹 브라우저에 전송하는 작은 데이터 조각(web site가 사용자의 브라우저에 남기는 작은 데이터 조각)
- 사용 예시 : 로그인 유지, 장바구니, 사용자 설정 기억
- 쿠키 동작 예시
1. 웹 서버가 요청된 페이지 + 쿠키를 포함한 응답을 브라우저에게 전송 -> 쿠키를 저장소에 저장하고, 쿠키의 속성(만료 시간, 도메인, 주소 등)도 같이 저장
2. 브라우저가 이후 같은 웹 서버에 웹 페이지를 요청하면 적용 가능한 쿠키를 포함하여 함께 전송
3. 웹 서버는 받은 쿠키 정보를 확인하고, 필요에 따라서 사용자 식별, 세션 관리를 수행 
4. 웹 서버가 요청에 대한 응답을 보내며, 필요 시 기존 쿠키 수정 or 새로운 쿠키 보냄.
- 쿠키를 KEY-VALUE 데이터 형식으로 브라우저가 저장 / Set-Cookie 필드를 통해 클라이언트에게 쿠키 전송
- 주요 용도 : 두 요청이 동일한 브라우저에서 들어왔는지 판단 / 상태가 없는 HTTP 프로토콜에서 상태 정보 기억


# 세션
- 서버 측에서 생성되어 클라이언트와 서버 간의 상태를 유지, 상태 정보를 저장하는 데이터 저장 방식(데이터를 **서버**쪽에 저장함. 클라이언트가 아님!)
- 고유한 세션 ID를 발급하여 이것을 쿠키에 담아 클라이언트에 보내 사용자를 식별함.
- 작동 원리
1. 클라이언트가 로그인 요청 후 인증에 성공 시 서버가 세션 데이터를 생성하고 저장함.
2. 세선 id 발급 및 이 id를 클라이언트에게 응답(데이터는 서버에 저장하고, 열쇠만 줌.) -> 클라이언트는 세션 id를 쿠키에 저장
3. 다시 동일한 서버에 클라이언트가 방문하면 세션 id가 저장된 쿠키를 서버에 전달
4. 서버가 id를 확인하고 로그인되어있다는 것을 확인함. -> 사용자의 요청을 처리하고 응답함.

# Authentication(인증)
- 필요성 : 클라이언트와 서버 간의 상태 정보를 유지하기 위해서 쿠키와 세션을 사용 / 클라이언트는 각기 다른 사용자를 식별해야 함.
- 다양한 방식이 존재(id and pw / social login / 생체 인증)
- django에서 기본적 뼈대 제공
### Django Authenticaion System
- django에서 사용자 인증과 관련된 기능을 모아 놓은 시스템
- User model, Session 관리, 기본 인증(Id/Password)등 다양한 기능 제공

### 기존 User Model의 한계
- 인증 후 사용되는 User Model은 별도의 클래스 정의 없이 내장된 auth 앱에 작성된 User클래스를 사용함.
- django의 기본 user model은 제공되는 필드가 매우 제한적
- 추가적인 사용자 정보 필요하면 이를 위해 기존 모델을 변경하기 어려움.
- INSTALLED_APPS = ['django.contrib.auth',] 부분에서 기본적으로 username, password, email 등의 필드를 가진 User 모델 제공함.
- 프로젝트의 특정 요구사항에 맞춰서 사용자 모델을 확장할 수 있음.

# Custom usermodel
- 사전 준비 : 두번째 app accounts 생성 및 등록
- auth 관련 경로를 내부적으로 accounts라는 이름으로 사용하기 때문에 되도록 app 이름은 accounts로 지정 / 
```python
# accounts/urls.py
app_name = 'accounts'
urlpatterns = []
# crud/urls.py
urlpatterns =[
  #...,
  path('accounts/', include('accounts.urls')),
]
```
1. Abstract 클래스를 상속받는 커스텀 User 클래스 작성(**기존 User클래스도 상속받기 때문에 커스텀 User클래스도 기존 User클래스와 완전히 같은 모습을 가진다.**)
``` python
# models.py에서 
from django.contrib.auth.models import AbstractUser

class User(AbstractUser):
    pass
```
2. django 프로젝트에서 사용하는 기본 User모델을 우리가 작성한 모델로 사용하도록 AUTH_USER_MODEL 값을 변경 (auth.User가 기본값)
```python
# settings.py

AUTH_USER_MODEL = 'accounts.User'
```
3. admin site에 대체한 User 모델 등록
```python
# accounts/admin.py

from django.contrib import admin
from django.contrib.auth.admin import UserAdmin
from .models import User

admin.site.register(User, UserAdmin)
```
- AUTH_USER_MODEL : django 프로젝트의 User를 나타내는데 사용하는 모델을 지정하는 속성 
**- 주의! : Django는 프로젝트 중간에 AUTH_USER_MODEL을 변경하는 것을 강력하게 권장하지 않음.** (이미 프로젝트가 진행되고 있으면 DB 초기화 후 진행.)
- 1,2,3을 거치면 커스텀 유저 모델이 DB에 반영됨.
- **기본 User모델이 충분하더라도 커스텀 User모델을 설정하는 것을 강력하게 권장함.

# Login
- 클라이언트와 서버 간의 상태 정보를 유지하기 위해서 쿠키와 세션 사용 -> 서버에 '나'임을 인증하는 과정이 Login
- 인증을 완료하고 세션을 만들고 클라이언트와 연결하는 것
1. 로그인
2. 로그인 인증
3. 세션 생성 후 서버 저장
4. 클라이언트에 쿠키 전달
### 로그인 기능 구현
1. 로그인 경로 url 및 링크 생성
```python
# accounts/urls.py

app_name = 'accounts'
urlpatterns = [
  path('login/', views.login, name='login'),
]
```
```html
<!-- articles/index.html -->

<h1>메인 페이지</h1>
<a href="{% url 'accounts:login' %}">Login</a>
```
2. url로 요청이 들어오면 실행할 함수 작성(AuthenticationForm사용)
```python
from django.contrib.auth.forms import AuthenticationForm

def login(request):
  if reqeust.method =="POST":
    pass
  else:
    form = AuthenticationForm()
    context = {
      'form': form,
    }
    return render(request, 'accoutns/login.html', context)
```
- AuthenticationForm() : 로그인 인증에 사용할 데이터를 입력받는 built-in form / ModelForm이 아닌 일반 Form을 상속받음.
3. 로그인 정보를 서버에 안전하게 전송하기 위해서 POST 방식 사용
```html
<!-- accounts/login.html -->
<h1>로그인</h1>
<form action="{% url 'accounts:login' %}" method="POST">
  {% csrf_token %}
  {{ form }}
  <input type="submit">
</form>
```
4. 사용자가 입력한 로그인 정보를 입력받는 정보를 입력받는 로직을 'POST'조건문에 작성
```python
from django.contrib.auth import login as auth_login

# 2번의 if request.method =='POST'부터 시작함.
if request.method =='POST':
  form = AuthenticationForm(request, request.POST)
  if form.is_valid():
    auth_login(request, form.get_user())
    return redirect('articles:index')
```
- login(request, user) : request는 사용자의 세션 정보에 접근하기 위해 / user는 어떤 사용자가 로그인 되었는지를 기록하기 위해
- get_user() : AuthenticationForm의 인스턴스 매서드 
-> 유효성 검사를 통과하면 로그인한 사용자 객체를 반환


# Template with Authentication data
- 템플릿에서 인증 관련 데이터를 출력하는 방법
- 현재 로그인 되어 있는 유저 정보 출력하기
```html
<!-- articles/index.html -->
<h3>Hello, {{ user.username }}</h3>
```
- context processors : 템플릿이 렌더링 될 때 호출 가능한 컨텍스트 데이터 목록 / django에서 자주 사용하는 데이터 목록을 미리 템플릿에 로드해둔 것
```python
# settings.py

TEMPLATES = [
  #...
  'context_processors':[
    #...
    'django.contrib.auth.context_processors.auth',
    #...
  ],
]

```
