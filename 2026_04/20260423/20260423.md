# 회원정보 수정
- User객체를 업데이트 하는 과정
1. 회원정보 수정 경로 url 생성
```python
# account/urls.py
app_name = 'accounts'
urlpatterns = [
  ...
  path('update/', views.update, name='update'),
]
```
2. 커스텀 유저 모델 사용하기 위해 Form 다시 작성
```python
# accounts/forms.py
from django.contrib.auth import get_user_model
from django.contrib.auth.forms import UserChangeForm

class CustomUserChangeForm(UserChangeForm):
    class Meta(UserChangeForm.Meta):
        model = get_user_model()
```
3. url 요청 들어오면 실행할 함수 작성
```python
# accounts/views.py

from .forms import CustomUserChangeForm

def update(request):
    if request.method == 'POST':
        pass
    else:
        form = CustomUserChangeForm(instance=request.user)
    context = {
      'form': form,
    }
    return render(request, 'accounts/update.html', context)
```
4. 작성 정보를 서버에 안전하게 전송하기 위해 "POST" 방삭 사용
```html
<!-- accounts/update.html -->

<h1>회원정보 수정</h1>
<form action="{% url 'accounts:update' %}" method='POST'>
  {% csrf_token %}
  {{ form }}
  <input type='submit'>
</form>
```
5. 메인 페이지에서 접근 가능한 태그 생성
```html
<!-- accounts/index.html -->
<a href="{% url 'accounts:update' %}">회원정보 수정</a>
```
- UserChangeForm 사용 시 문제점 : User모델의 모든 정보들까지 출력됨. / 일반 사용자들이 접근하면 안되는 정보까지 출력
- CustomUserChangeForm 출력 필드 재정의

```python

# accounts/forms.py
from django.contrib.auth import get_user_model
from django.contrib.auth.forms import UserChangeForm

class CustomUserChangeForm(UserChangeForm):
    class Meta(UserChangeForm.Meta):
        model = get_user_model()
        fields = ('first_name', 'last_name', 'email')
```
```python
# accounts/views.py

from .forms import CustomUserChangeForm

def update(request):
    if request.method == 'POST':
        form = CustomUserChangeForm(request.POST, instance=request.user)
        if form.is_valid():
            form.save()
            return redirect('articles:index')
    else:
        form = CustomUserChangeForm(instance=request.user)
    context = {
      'form': form,
    }
    return render(request, 'accounts/update.html', context)
```
# 비밀번호 변경
- Session 데이터를 update하는 과정
- 기능 구현 작성
1. django는 비밀번호 변경 페이지를 Set password로 안내함(기본 경로는 /accounts/password/)
2. Django에서 안내하는 비밀번호 변경 URL에 맞춰서 작성
```python 
# accounts/urls.py

app_name = 'accounts'
urlpatterns = [
  ...,
  path('password/', views.password, name='password'),
]
```
3. url로 요청 들어오면 실행할 password 함수 사용
```python
# accounts/views.py

from django.contrib.auth.forms import PasswordChangeForm

def password(request):
    if request.method == 'POST':
        form = PasswordChangeForm(request.user, request.POST)
        if form.is_valid():
            user = form.save()
            return redirect('articles:index')
    else:
        form = PasswordChangeForm(request.user)
    context = {
        'form': form,
    }
    return render(request, 'accounts/password.html', context)
```
4. html 작성
```html
<!-- accounts/change_password.html -->
<h1>비밀번호 변경</h1>
<form actions="{% url 'accounts:password' %}" method="POST">
    {% csrf_token %}
    {{ form }}
    <input type="submit">
</form>
```
### 세션 무효화 방지
- 암호 변경 시 세션 무효화를 막아주는 함수 : "update_session_auth_hash"
```python
# accounts/views.py

from django.contrib.auth import update_session_auth_hash
from django.contrib.auth.forms import PasswordChangeForm

def password(request):
    if request.method == 'POST':
        form = PasswordChangeForm(request.user, request.POST)
        if form.is_valid():
            user = form.save()
            update_session_auth_hash(request, user)
            return redirect('articles:index')
    else:
        form = PasswordChangeForm(request.user)
    context = {
        'form': form,
    }
    return render(request, 'accounts/password.html', context)
```

# 비밀번호 암호화
- 암호화의 중요성 : 많은 해킹상태가 발생하고 있음.
1. 사용자가 입력한 비밀번호 그대로 저장하는 방식은 보안에 취약하므로 X
2. 일정한 규칙에 따라 비밀번호를 알아볼 수 없는 문자로 '인코딩'한 후 저장 -> 보안에 매우 취약하므로 X
---
3. **비밀번호를 복원이 불가능한 고정된 길이의 문자열로 변환 후 저장 -> 보안에 필수!**
### 해시
- 비밀번호를 복원이 불가능한 고정된 길이로 바꾸는 과정
- 임의의 크기를 가진 데이터를 고정된 크기의 고유한 값으로 변환하는 것
- 이를 해주는 함수를 해시함수라고 함.
- **Django는 기본적으로 SHA-256 해시 함수를 사용해서 암호화**
#### SHA-256
- 현재 가장 널리 사용되는 표준 암호화 해시 알고리즘
- 어떤 데이터를 넣어도 항상 256비트(64글자)의 결과물을 생성
---
#### 레인보우 테이블
- 공격자가 자주 사용되는 비밀번호들을 수백만, 수십억 개를 해시로 변환해 저장해둔 거대한 정답지
- 레인보우 테이블 공격 방식
1. 공격자가 DB를 탈취해 사용자의 비밀번호 해시 값을 얻음
2. 해시 값을 자신의 레인보우 테이블에서 검색
3. 일치되는 값을 찾아내면서, 역방향으로 돌리지 않고 비밀번호 알아내는 데 성공
---
- 이에 대한 해결방안 : 같은 비밀번호라도 사용자마다 임의의 문자열(솔트)을 비밀번호에 붙여서 해시 암호화 진행
---
#### 솔트
- 각 사용자마다 고유하게 생성된 임의의 문자열
- 비밀번호에 덧붙여서 해시 값 생성
---
#### 무차별 대입 공격
- 가장 원시적이지만 강력한 방법
- 가능한 모든 비밀번호를 하나씩 대입
- 시간이 오래 걸리지만, 현대 컴퓨터의 발전으로 빠른 연산 속도 가능
- 이에 대한 해결방안 : 연산 속도를 늦추기 위해 의도적으로 비밀번호 검증 과정을 느리게 함. -> 키 스트레칭
---
#### 키 스트레칭
- 솔트를 적용한 해시 함수를 수만~수십만 번 반복하여 연산 시간을 의도적으로 늘리는 기법
- 무차별 대입 공격 방어 가능
- Django는 이 키 스트레칭을 구현하기 위해 PBKDF2라는 검증된 알고리즘 사용
- 1개 비밀번호 검증시간 : 키 스트레칭이 없으면 약 0.000001초(연산 1회)라서 10억 개 비밀번호 검증이면 11.5일 이지만 키 스트레칭 적용되면 약 0.1초(연산 39만회)라서 10억 개 비밀번호면 약 3,170년 소모
- PBKDF2 : <algorithm>&<iterations>&<salt>&<hash> 어떤 알고리즘 쓰는 지, 키 스트레칭 횟수, 생성된 솔트, 생성된 최종 해시

# (참고)비밀번호 초기화
- 비밀번호 잊어버린 사용자가 이메일을 활용하여 비밀번호 다시 설정하는 과정
1. 비밀번호를 찾으려고 하는 이메일 입력
   1. Django에서 제공하는 비밀번호 관련 기능을 활용하기 위해 urls.py 수정
   ```python 
   # crud/urls.py

   from django.contrib import admin
   from django.urls import path, include

   urlpatterns = [
       path("admin/", admin.site.urls),
       path('articles/', include('articles.urls')),
       path('accounts/', include('accounts.urls')),
       ...,
       path('accounts/', include('django.contrib.auth.urls')),
   ]
   ```
   2. 비밀번호를 찾으려고 하는 이메일 입력
   - '.../accounts/'만 입력해보고 제공하는 url 목록 확인
   - 제공되는 url목록에서 'password_reset'경로를 입력해보고, 비밀번호 초기화 페이지로 접근
2. 이메일로 비밀번호 재설정 링크를 전송
   1. 비밀번호 찾으려는 이메일 작성하고 "Reset my password"버튼을 눌러 이메일 전송
   2. 이메일 관련 설정을 하지 않았기 때문에 오류뜨고 전송 실패
   3. Django에서는 이메일을 보낸 내용을 콘솔에서 볼 수 있는 기능 제공
   ```python
   # crud/settings.py
   EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'
   ```
   - 세팅 후 다시 이메일 전송하기
   4. 콘솔창에 비밀번호 초기화 설정 페이지 안내 이메일을 확인하고, 해당 링크 들어가기
3. 비밀번호 재설정 페이지에서 새로운 비밀번호 설정
- 설정 후 비밀번호 설정 완료 페이지 보여주기
4. 초기화 후 다시 로그인
