# Model

- model : DB와 python 클래스(객체)로 추상화된 형태로 상호작용
- 개발자가 DB에 대한 깊은 지식 없이도 쉽게 데이터 관리 가능

- urls.py : 사용자 요청의 시작점
- views.py : 요청을 처리하고 models.py를 통해 데이터를 다룸
- models.py : DB를 정의하고 DB와 상호작용
- templates : views.py로부터 받은 데이터를 사용자에게 보여줄 화면을 구성

### Model class
- DB의 테이블을 정의하고 데이터를 조작할 수 있는 기능들을 제공
- 상속받아서 DB를 정의함.
```python 

class Article(models.Model): # 반드시 (models.Model)을 상속받아야 함.
    title = models.CharField(max_length=10) # max_length를 통해서 길이 제한 가능
    content = models.TextField() # 내용을 미리 정하고 쓰지 않기 때문에 길이 제한을 딱히 걸어놓지 않음. 이런 상황에서는 TextField를 사용
    # title은 말 그대로 제목, content는 콘텐츠(내용)
    # id 필드는 Django가 자동으로 생성함.
```
- Model은 model에 관련된 모든 코드가 이미 작성되어 있는 클래스 -> 개발자는 가장 중요한 테이블 구조를 어떻게 설계할지에 대한 코드만 작성하기 위한 것이다.
- 클래스 변수명은 테이블의 각 "필드(열) 이름"

###### TIP(참고)
- Field는 DB의 열을 나타내는 중요한 구성 요소 -> 데이터의 유형과 제약 조건을 정의함.
- max_length를 지정하는 것은 선택 사항이고, SQLite는 문자열 컬럼의 길이에 대한 엄격한 제한을 강제하지 않음. 
- 그러나 유효성 검사와 컬럼에 들어갈 데이터를 명확하게 하기 위해 설정하는 것은 권장된다.

### Model Field
- DB 테이블의 필드(열)을 정의
- 데이터 타입 및 제약 조건을 명시함.(유효성 검사와 안정성 증가)


### Field Options
- 필드의 '동작'과 '제약 조건'을 정의
- 주요 필드 옵션 : null(DB에서 null값을 허용할 지 여부 결정), blank(form에서 빈 값을 허용할지 여부 결정), default(필드의 기본값을 설정)
##### 제약 조건
- 특정 규칙을 강제하기 위해 테이블의 열이나 행에 적용되는 규칙

# Migrations
- model 클래스의 변경사항(필드 생성, 수정 삭제 등)을 DB에 최종 반영하는 방법
- migrations 폴더를 통해 DB의 수정 및 변경 사항을 추적할 수 있다.
- migration 파일 == 최종 설계도와 같다.(이 방식을 이용하여 데이터베이스를 구축할 것)

- python manage.py makemigrations -> model class를 기반으로 최종 설계도(migration)을 작성
- python manage.py migrate -> 0001_initial.py를 통해서 실제 데이터베이스가 생성된다.

- 기본 형식 : (앱이름)_(모델명) --> auth_ : 회원, 권한에 대한 정보(사용자 정보) / articles_article : id, title, content를 담고있는 테이블 생성 


### 요약
- models.py에서 설계도 초안을 작성하고, -> migration 파일(0001_initial.py)로 보낸 뒤에 migrate를 작성하면 -> dv.sqlite3라고 DB가 생성된다.

# 주의
- migration 폴더에 자동으로 생성된 파일들은 직접 수정하거나 삭제하지 않는 것이 원칙이고, 차라리 새로 만드는 게 편함.
- 이미 생성된 테이블에 필드를 추가해야 한다면? -> 이미 기존 테이블이 존재하는 경우, 새 필드를 추가할 때 문제 발생 가능


### DateTimeField의 필드 옵션
- auto_now : 지금 시간을 자동으로 저장
- auto_now_add : 데이터가 처음 생성될 때만 자동으로 현재 날짜 시간을 저장

###### 로그인
- python manage.py createsuperuser : 로그인 정보 설정
- admin.py에서 admin.site.register(Article) -> article 등록 가능


# 데이터베이스 초기화 방법
1. migration 파일 삭제(폴더는 삭제하는 것이 아님!), __init__.py와 migrations 폴더는 삭제 금지.
2. db.sqlite3 파일 삭제
  
### 기타 명령어
- python manage.py showmigrations : migrations 파일들이 migrate되었는지 여부를 확인함.
- python manage.py sqlimgrate articles 0001
