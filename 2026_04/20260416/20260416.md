# Django Form

### HTML form
- 사용자로부터 데이터를 제출 받기 위해 활용한 방법이나 -> 비정상적 혹은 악의적인 요청을 필터링 할 수 없다.

### 유효성 검사
- 수집한 데이터가 정확하고 유효한지 확인하는 과정(입력한 데이터가 올바른 형식인지 자동으로 점검하는 기능 제공)
- 직접 개발하는 것이 아니라 django가 제공하는 form 사용

# django form
- 사용자 입력 데이터를 수집하고, 처리 및 유효성 검사를 수행하기 위한 도구
- form class 정의 : form class를 상속받아 내용과 제목에 대한 사용자 입력을 받는 ArticleForm을 정의하는 방법

### Form Class를 적용한 new logic
- view함수 new 변경
- from = ArticleForm() # 빈 인스턴스를 전달받음.
- new.html에서 input ~~ 없이 {{ form }}만으로도 운영 가능

# Widgets
- HTML input 요소의 표현을 담당.
- content = forms.CharField(widget=forms.Textarea) -> 입력을 textarea형태로 변환함.

### Form vs ModelForm
- Form : 사용자 입력 데이터를 DB에 저장하지 않을 때(ex. 검색, 로그인)
- ModelForm : 사용자 입력 데이터를 DB에 저장해야 할 때(ex. 게시글 작성, 회원가입)

# ModelForm
- Model과 연결된 Form을 자동으로 생성해주는 기능을 제공, Form 클래스 + Model 클래스
- Meta클래스 : model 정보와 fields 정보를 필수로 요구함.
- field에는 튜플, list 형태 들이 들어갈 수 있고, 모두를 지정하고 싶으면 field = '__all__'를 입력하면 됨.
