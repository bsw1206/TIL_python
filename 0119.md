## 타입
- 변수나 값이 가질 수 있는 데이터의 종류

## data type
- 값의 종류와 그 값으로 할 수 있는 동작을 결정하는 속성
- 필요한 이유 : 각 타입에 따라 가능한 기능과 연산이 다르기 때문이다.

---

### 숫자형 데이터
- 프로그래밍에서 값을 계산하고, 수량을 세고, 데이터를 분석하는 등 가장 기본이 되는 타입
- ex. 정수형(int), 실수형(float)...

##### 정수형 데이터
- 소수점이 없음
##### 실수형 데이터
- 소수점이 있음

##### 지수 표현법
- 아주 크거나 아주 작은 실수를 간결하게 표현하는 방식
``` python
1230000000 (1.23 * 10^9)
big_number = 1.23e9
1230000000 (1.23 * 10^-9)
small_number = 1.23e-9
```
##### 산술 연산자
- 산술 연산자 : 가장 기본이 되는 연산(+, -, *, /, //, %, **, -)
- 연산자 우선순위 : **(높음) , 음수 부호, 곱셈/나눗셈/정수/나눗셈/나머지 , 덧셈
---
- Quiz
```python
print(-2 ** 4) # -16 -> 우선순위가 거듭제곱이기 때문에
print(-(2 ** 4)) # -16
print((-2) ** 4) # 16
```

### Sequence Types
- 여러 개의 값들을 순서대로 나열하여 저장하는 자료형
- 시퀀스 : 여러 데이터가 정해진 **순서**대로 일렬로 늘어선 자료 구조
- ex. str, list, tuple, range
- 인덱스(index) : 시퀀스 자료형에서 각 값의 위치를 식별하기 위해 부여된 고유한 번호
#### **시퀀스 타입의 5가지 공통 특징**

> - 순서 : 값들이 순서대로 저장(**정렬을 의미하는 것은 아님!!**)
> - 인덱싱 : 각 값에 고유한 고유 번호(인덱스)를 가지고 있으며, 이를 사용하여 특정 위치의 값을 수정 가능
> - 슬라이싱 : 인덱스 범위를 조절하여 전체 데이터 중 원하는 부분만 값을 잘라내서 사용할 수 있음.
> - 길이 : len() 함수를 사용하여 저장된 값의 개수(길이)를 구할 수 있음.
> - 반복 : 반복문을 사용하여 각 값을 하나씩 순서대로 꺼내 사용할 수 있음.

---
##### 시퀀스 타입 특징 예시
my_data = "Hello"라는 문자열 데이터가 있을 때
> **문자열도 sequence다!!**
- my_data[1] = 'e'
- my_data[1:4] = 'ell'
- len(my_data) = 5

###### 문자열
- 문자들의 순서가 있는 변경 *불가능*한 시퀀스 자료형
- 만드는 법 : 작은따옴표, 큰따옴표로 감싸서 표현
```python
print('Hello World!') # Hello World
print(type('Hello World!')) # str
```
---
**문자열 안에 큰따옴표/작은따옴표를 사용하려면 작은따옴표/큰따옴표로 묶는다.

##### 이스케이프 시퀀스 예약 문자

- \n : 줄 바꿈
- \t : 탭
- \\ : 백슬래시
- \' : 작은 따옴표
- \" : 큰 따옴표

#### f-string
- 문자열 시작 전 'f' 접두어를 붙이고 삽입할 부분을 {}로 감싸줌. 
```python
name = '백승우'
age = 26
greeting = f'안녕하세요, 제 이름은 {name}이고 나이는 {age}살입니다.' # 안녕하세요, 제 이름은 백승우이고 나이는 26살입니다.
print(greeting)
```

##### 인덱스
- 시퀀스 자료형에서 각 값의 위치를 식별하기 위해 부여된 고유한 번호
- *1이 아닌 0부터 시작한다는 것은 거리의 개념으로 이해하면 쉽다!*
- 시작점에서 얼마나 떨어져 있는가를 의미한다.
- index 0 : 첫 번째 항목
- "hello" = h의 인덱스 : 0 or -5, e의 인덱스 : 1, -4 .... o의 인덱스 : 4 or -1

### 슬라이싱 사용 방법
> my_sequence[start:stop:step]
> - start : 슬라이싱을 시작할 인덱스(포함됨)
> - stop : 슬라이싱을 끝낼 인덱스(포함되지 않음!!!)
> - step : 몇 개씩 건너뛰며 값을 가져올지에 대한 간격
> - 필요에 따라서 생략할 수 있음.(start 생략 : 처음, end 생략 : 끝, step 생략 : 1)

###### 참고
- 0b : 2진수, 0o : 8진수, 0x : 16진수

- 실수의 함정
- ex. 
  ```python
  result = 0.1 + 0.2
  print(result == 0.3) # False
  print(result) # 0.300000000000...0004
  ```
  > 이유 : 
  > - 컴퓨터를 모든 숫자를 2진수로 변환하기 때문에 우리가
  > - 사용하는 10진수에서 0.3을 2진수로 변환하면 무한하게 나열됨.
  > - 컴퓨터는 최대한 근삿값을 나타내게됨.
  > - 해결책 : 실수를 계산할 때에는 함수를 사용함.
