# 선형 회귀
- 입력과 출력 사이의 관계를 직선 형태로 구사, 지도 학습의 기초
- 단순 선형 회귀 : 하나의 설명변수와 하나의 반응 변수 사이의 선형 관계를 찾는 방법 / 목표 : 데이터를 가장 잘 설명하는 직선을 찾아 예측에 활용함.
- 단순 선형 회귀의 모형 : Y = b0+b1X + 엡실론

# 최소제곱법
- 실제 관측값과 예측값의 차이(잔차)를 제곱해 합한 값을 최소화하는 방법
- 목표 : 데이터를 가장 잘 설명하는 직선을 찾기 위해 계수를 추정함.
- 잔차 : 오류랑 비슷, 실제값과 예측값의 차이

# 단순 선형 회귀
- 목표 : TV광고비(X)와 제품 판매량(Y)의 선형 관계 예측
- 계수 해석 : 절편 = 7.03 -> 광고비가 0이어도 기본적으로 평균 판매량이 7.03백만원이다
- X의 계수 =0.0475 -> TV광고비를 1단위 늘리면 평균 매출이 약 0.0475*1단위=4.72만원 증가
- 모형 적합도 (R^2이 높을수록, 1에 가까울 수록 좋음.)

### 잔차제곱합(중요)
- 모델의 예측값과 실제 데이터 값의 차이(잔차)를 제곱하여 모두 더한 값으로 ,오차의 총합을 의미함.


# 다중 선형 회귀
- 독립 변수가 여러 개 존재할 때 사용하는 회귀 분석 기법
- 여러 변수를 동시에 고려하여, 종속 변수와의 관계를 파악
- Y = b0 + b1X1 + b2X2 + ... + bpXp + 엡실론
- 추정 방법 : 실제 값과 예측 값의 차이를 제곱해 합한 값(RSS)을 최소화
- 행렬을 표현하고, 최소 제곱법을 하고, 정규방적식(closed-form 해)를 구함

- 결과 해석 : 모형 적합도는 R^2으로 구함. 단순 선형회귀에 비해 높은 결정계수 값을 가진다.

# 선형 회귀 주의사항
- 훈련 데이터에서의 성능, 테스트 성능 평가 필요(과적합 문제)
- 이상적인 상황 : 변수들이 연관되어 있지 않을때 <-> 연관되어 있다면 계수 추정이 불안정해지고 혼동 발생
- 관찰 데이터의 상관관계로 인과 관계를 주장해서는 안 됨.

# 분류
- 정해진 범주 중 하나로 지정함.
- 목표 : 분류 함수 학습 -> 입력 X가 속할 범주(카테고리)예측 (예: 신용카드 연체)
- 분류 문제에 선형 회귀를 써도 될까? 되겠냐? -> 예측 확률이 0보다 작거나 1보다 크게 예측될 수 있어 쓰기 부적절함.
- 적합한 모델 : 시그노이드 함수

# 로지스틱 회귀의 모형식
- 시그모이드 함수 : y = 1/(1+e^(-z))
- 로지스틱 함수와 선형회귀의 관계 : 로지스틱 함수는 선형 함수 내포함.
- 로지스틱 모형 : 선형 모형식 + 시그모이드 함수
- 우도 : 모수(b0, b1)를 추정하는 방법 / 현재 확률 함수가 데이터를 얼마나 잘 설명하는지 나타낸 지표
- 우도를 최대화하는 것 : log변화한 likelihood, 그냥 likelihood랑 똑같! 미분하면서 도함수=0에 근접하도록 b를 찾아나감.
- 결과 해석 : 입력 값 증가에 따라서 예측 확률 값이 기하급수적으로 증가할 수 있음.(비선형이라서) / 여러 변수를 함께 모형에 사용하면 회귀 계수를 통해 연체할 확률을 계산할 수 있다.

# 단순 선형모델(1D) : 모수적 함수
- 손실 함수 -> 최소제곱 손실함수
- 순차적으로 모수를 조정하면서 찾아내는 선형 회귀

# shallow 네트워크
- 입력층 - 은닉층 1개 - 출력층으로 구성된 1개의 은닉층만 가진 신경망 구조
- 복잡한 비선형 패턴 해석가능
- 단순 선형 회귀보다 식이 좀 길어짐(모수에 새로운 값(또 다른 모수)을 곱함.)
- 활성화 함수 : ReLU(Rectified Linear Unit, 활성화 함수의 대표적인 예시)가 사용됨.
- 모수 : 훈련데이터가 주어지면 손실 함수를 정의하고 손실을 최소화하도록 모수를 조정함.
- piecewise linear 함수 : 입력 구간을 나눠 조각별 선형 함수를 만든다.
- Hidden Units : 추가적인 모수를 감춘다. (다수의 모수를 하나의 변수로 묶는다.)
- 선형 함수 -> 활성화 함수를 적용(선형에서 특정 조건에서 증가하는 모양) -> 활성화 후 (?)배 적용하면 음수 방향으로 꺾임. -> 합치면 최종 함수가 나온다.

### hidden units
- 충분히 많은 hidden unit이 있다면 임의의 1차원 함수를 원하는 정확도로 근사할 수 있다.

# 다중 출력/입력
- 2개 출력의 네트워크? 각각에 대한 출력 함수가 나옴. 근데 두 함수의 꺽이는 위치가 같음.(hidden unit을 공유한다면!)
- 선형 변환 : hyperplane이 나타남. -> 활성화함수를 적용하면 그냥 평면. -> 히든 유닛에 상수배 적용하면 다시 초평면 등장. -> 합치면 접힌 구조가 나타난다.

# Deep 네트워크
- 여러 개의 hidden layer를 가짐.
- shallow 네트워크와 deep 네트워크 비교? 딥 네트워크의 표현력이 더 높다. 
- 합성을 할 수록 복잡하게 접힌 구조가 발생함.
- 수식 표현 : 각 네트워크의 hidden unit을 새로운 파라미터를 사용하여 단순화시킴.
- shallow의 계산 과정이랑도 비슷함.

# 1D 선형회귀 예시(중요)
- 등고선, 데이터와 선형함수 직선
- 경사 하강법 : 손실함수의 값이 줄어드는 방향으로 파라미터를 이동하는 과정 (등고선에서는 어두운 쪽으로, 선형함수에서는 최적의 선을 찾는 방향으로)
- 미분을 통해서 최적화 가능!
- 경사 하강 알고리즘 -> 손실 함수를 최소화하기 위해 파라미터를 반복저긍로 갱신하는 알고리즘 / 기울기 계산? -> 손실 함수를 파라미터에 대해 편미분 진행 -> 파라미터 업데이트
- 경사 하강법의 과정 : 데이터 -> 손실함수 3D 그래프 -> 손실함수 등고선(2D) -> 선형함수의 업데이트
- 경사 하강법의 단점 : nonconvex 문제에서 지역의 최소점에 빠지기 쉬움. -> 해결 방안 : 매 스텝마다 전체 데이터에 대한 미분값을 구하여 업데이트함. 스텝별 계산량이 늘어나게 됨.
- 대안 : 무작위 확률로 샘플된 일부 데이터(batch)만 사용하여 기울기 계산(SGD, 확률적 경사 하강법)
