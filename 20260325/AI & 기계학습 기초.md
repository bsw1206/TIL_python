# AI, ML, DL의 정의 / 데이터 구성 요소 / 회귀 설명력 : R^2(결정계수) / 혼동행렬 / 회귀

# AI, ML, DL(중요)
- AI : 가장 넓은 단위, 주어진 환경 데이터를 인지, 학습, 추론을 통해서 목표 달성을 하도록 계획하는 시스템
- ML : 데이터로부터 학습하여 목적을 달성하는 접근 방법(예 : 생성형 AI, 언어 모델, 이미지 분류 모델, 추천 시스템)
- DL : ML 범주 내에서 신경망 함수를 사용한 학습 방법론
- ML이 아닌 AI : 규칙 기반 시스템, 휴리스틱 기반 알고리즘

# 데이터 구성요소(중요)
- 데이터의 중요성 : 머신러닝에서 규칙을 직접 학습시키지 않고 데이터에서 규칙을 학습
- feature : 모델이 예측에 사용하는 입력 정보(판단의 근거)(예 : 영상의 정보, 메일제목과 발신자)
- label : 모델이 예측하려는 정답(목푯값)(예 : 영상에 대한 사용자 피드백, 스팸/정상 여부)

# 1차원 기반 피쳐 학습
- 특징이 하나일때(가장 단순한 형태)
- income = f* + 엡실론 (f*는 미지의 참함수, 관측할 수 없다., 엡실론은 측정 오차)
- f*는 어차피 관측할 수 없기 때문에, 어떤 f가 더 피쳐와 라벨의 관계를 잘 나타낼 수 없는 함수라고 특별히 지칭하기 어렵다.

# 모델과 가설 공간(중요)
- 학습 : 입력 -> 출력의 관계, 하나의 함수로 표현(그러나 관계를 나타내는 함수는 무수히 많음.)
- 가설 공간 : 관계를 표현할 수 있는 모든 후보 함수의 모음
- 모델 : 가설공산에 속한 특정 함수
- 파운데이션 모델 : 대규모 데이터를 사전 학습하여 다양한 작업에 응용할 수 있는 기초가 되는 모델

# 학습
- 주어진 데이터가 정답을 잘 맞출 수 있도록 모델의 규칙을 조금씩 조정해가는 과정
- 데이터 -> 가설공간 -> 선택 기준에 대한 선택된 모델


**- 학습 과정 : 가설 공간에서 함수 선택 -> 모든 예시예측 -> 손실 함수 계산 -> 더 적게 틀리도록 함수의 파라미터 조정 -> 반복하여 최적의 모델 완성(중요!!)**


# 2D 피쳐 기반 학습
- income = f* + 엡실론(income : 우리가 예측하려는 라벨 , 주로 Y로 표기)
- 파란색 표면은 관측 불가능(f*)
- 빨간색 점들(데이터)만 관측 가능함.
- 학습 전 : 어떤 가설 공간을 사용하는가?
- 학습 후 : 데이터를 활용하여 어떤 모델을 선택하는가?
**- Y = f*(X) + 엡실론 (엡실론은 피쳐 X와 독립이고, 규칙성이 없다.)(X는 많은 값이 존재한다.)(중요!!)**

### 왜 f(.)를 학습하는가?
- 잘 학습된 f가 있으면 새로운 입력에서 Y 예측 가능
- 어떤 X가 Y를 설명하는데 중요하고 어떤 것은 관련이 적은지 알 수 있다.

# 지도 학습
- 입력, 정답을 가지고 예측 규칙을 배우는 방법
- 처음 보는 데이터에서도 예측 성능 향상을 위해서
- 데이터 : 입력과 정답이 항상 쌍으로 존재(예 : 고양이의 모습을 학습하고 다른 고양이를 만났을 때 고양이를 예측)
- 종류 : 회귀, 분류
- 회귀 : 예측하고 싶은 결과값이 숫자일 때 사용함.(가격, 점수, 온도), (특징 : 라벨 및 예측의 출력은 연속적인 수치다.)
- 분류 : 예측하고 싶은 결과값이 범주일 때 사용(스팸/정상, 질병 유무)

### 지도학습 용어
- 특성 : 예측에 사용되는 설명 변수(input값)
- 라벨 : 맞춰야 하는 정답(실제값)
- 예측값 : 모델이 내놓은 결과(숫자 or 범주)
- 오류 : 라벨과 예측값의 차이(오류를 줄이는 것이 지도학습의 목표!!!)

# 회귀 오류 : 평균제곱오차(MSE)(중요)
- 1/n(시그마)(라벨-예측값)**2
- 각 데이터에서 정답과 예측의 평균 제곱 차이값
- 틀린 정도를 제곱해서 더 큰 오류에 패널티 부여 -> 전체 평균 오류 수준을 한눈에 볼 수 있음.
- 크기 범위 : 0에서 무한대
- 실제 데이터 단위의 제곱을 사용, 0에 가까울 수록 좋음, 구체적 오차 크기 알 수 있음, 직관적 해석 어려움

### 회귀 설명력 : 결정계수 R^2(중요)
- 라벨의 분산 중에서 피쳐로 설명되는 비율
- 1에 가까울수록 설명력이 높고, 낮을수록 설명력이 낮음
- 분자는 내가 조절 가능, 분모는 내가 조절 불가능
- 1보다 클수는 없음, 음수도 나올 수 있음.(예측값들이 평균값 보다도 못한다면, 나쁜 모델!)
- 1에 가까울 수록 좋음, 직관적 해석이 가능함, 구체적 오차 크기 알 수 없음

## MSE와 R^2는 상호 보완적이다.


# 분류 문제 :
- 입력 특성으로부터 출력값이 범주값으로 나타나는 문제
- 특징 : 범주 라벨(이진/다중)(이진 예 : 스팸 파악, 다중 예 : 학점 예측)
- MSE나 R^2를 사용하기 어려움.

### 정확도
- 전체 예측 중에서 올바르게 맞춘 예측의 비율
- 예측 범주와 실제 데이터가 조건이 같으면 1 아니면 0 -> 적중률을 계산한 것임.(0과 1사이의 값)
- 정확도가 아무리 맞다고 해도 좋은 모델이 아닐수도 있음.
- 불균형 데이터 문제로 인해서 단순히 비율만 보게 됨. 다른 지표도 필요함

### 혼동 행렬(중요)
- 예측과 실제 값 사이의 관계를 행렬 형태로 표현
- 예시 : TP(실제 양성, 예측도 양성),TN(실제 음성, 예측도 음성),FP(실제 음성, 예측 양성, 오탐),FN(실제 양성, 예측 음성, 누락)
- 정밀도 : 양성이라 판정한 것 중 진짜 양성의 비율 = TP/(TP+FP)
- 재현율 : 진짜 양성 가운데 잡아낸 예측 양성 비율 = TP/(TP+FN)
- F1-score : 정밀도와 재현율의 조화평균, 2*(정밀도*재현율)/(정밀도+재현율), 높을 수록 좋음.

### 오버피팅
- 훈련 데이터의 우연한 패턴까지 외워버려서 훈련에서는 잘 맞지만 테스트에서는 성능이 나빠지는 현상
- 현상 : 현상 오류 급격히 낮음, 테스트 오류 높음/요동
- 오버피팅과 분포 변화가 다름으로 인한 에러 증가
- 분포 변화로 인한 에러 증가는 모델이 과적합하지 않아도 발생 가능하다.
- 오버피팅 vs 언더피팅 : 언더피팅은 모델이 단순하거나 학습이 완료되지 않은 상태.(중요한 패턴을 놓침, 오류가 크다) -> 해결 실마리 : 더 많은 데이터, 테스트 데이터를 활용한 모델 선정, 교차 검증

# 테스트 성능 평가
- 훈련 오류 : 모델을 학습시킨 같은 데이터에 다시 적용해 계산한 오류(문제집 접수)
- 테스트 오류 : 학습에 쓰지 않은 새 관측치에 대해 모델을 적용했을 때의 평균 예측 오류(시험 점수)
- 두 개는 다르고 훈련 오류는 테스트 오류를 과소평가 하는 경우가 많음.
- 모델이 복잡해질수록 훈련 오류의 예측 오류는 감소, 테스트 오류는 감소하다가 증가(중간 지점이 가장 적합.)
- 테스트 오류는 현실에서 구하기 어려움 -> 데이터를 나눠 여러 번 훈련과 평가를 반복하여 테스트 오류를 가늠한다.(검증셋, K겹 교차검증)

### 검증셋
- 가용 샘플들을 무작위로 훈련셋과 검증셋으로 분할 -> 학습은 훈련셋, 성능평가는 검증셋에서 진행(단점 : 들어가는 표본에 따라서 추정치가 매우 가변적이다 -> 예측 성능이 낮아질 수 있음.)

### K겹 교차검증
- 테스트 오류 추정의 표준적 접근(문제집을 5등분해서 서로 다른 모의고사를 5번 보는 것)
- 데이터를 셔플하여 K개의 폴드로 섞이지 않게 무작위 분할
- K개의 MSE 평균해 테스트 오류를 측정
- 1~n 까지 총 K개에서 K=n이면 leave-one-out 교차검증 -> 그럼 여기서 검증셋의 크기는? 1이다. 검증셋이 1이면 훈련셋은 그걸 뺸 전부 -> 어차피 전부를 검증하므로 굳이 셔플할 필요가 없다!

# 비지도 학습
- label없이 데이터의 구조, 패턴, 집단을 찾아내는 학습
- 대표 과제 : 군집화, 차원 축소, 밀도추정/이상치 탐지
- 핵심 : 무엇을 비슷함, 다름으로 볼 것인가? / 전처리를 어떻게 할 것인가?
- 비지도 학습 vs 지도 학습 : 지도학습은 입력 + 라벨 / 비지도 학습은 입력만으로 구조 학습, 클러스터링(서로 비슷한 데이터끼리 묶음) 존재

# 클러스터링
- 데이터 안에서 하위 집단을 찾는 기법들의 총칭
- 집단 내부는 서로 유사, 집단 간은 상이하도록 데이터를 분할
- K 평균, 계층적 군집의 두 가지 대표 클러스터링 기법 
- 주의점 : 스케일링(표준화가 필요한가?), 몇 개의 클러스터가 적합한지?, 여러 번의 시도를 권장함.



### K mean clustering
- K를 정해놓고 분류
- 점들의 색상이 갖는 의미는 없고 그냥 다른 클러스터라는 것만 알 수 있다.
- 모든 클러스터의 합집합은 전체의 개수(각 클러스터는 적어도 하나의 군집에 속함.)
- 교집합은 존재하지 않는다. / 정확히 하나의 군집에만 나눈다.
- 클러스터를 부여하고 다음과 같은 과정을 반복(각 클러스터의 중심 계산하고, 각 관측치를 가장 가까운 중심의 클러스터에 재할당)
- 매 반복마다 결과는 더 좋아지지만 항상 최적의 해를 갖는 것은 아니다. 초깃값에 따라서 달라짐.

### 계층적 군집 알고리즘
- 데이터들이 점차 큰 클러스터로 합쳐지는 과정
- 매 단계에서 클러스터끼리의 병합 이루어짐. / 1개의 단일 클러스터가 될 때까지 진행됨.
- 링크의 유형 : single(최소 거리) 링크(쌍별 거리 중 최소값), complete(최대 거리) 링크(쌍별 거리 중 최대값), average(평균 거리) 링크(쌍별 거리 중 평균)
- 링크에 따라서 결과가 달라짐.
