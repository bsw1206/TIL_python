# 디코딩 알고리즘, 거대 언어 모델의 평가, 선호 학습

# 파운데이션 모델
- 대량의 데이터를 기반으로 사전 학습된 대규모 AI 모델
- 다양한 작업에 범용적으로 활용할 수 있는 기초 역할을 함.
- 예시 : 텍스트 생성, 비디오 생성, 멀티 모달 입력을 통해 멀티모달 출력
- 자세한 설명(프롬프트)를 입력하여 새로운 테스크 해결
- SORA : 비디오 파운데이션 모델, ChatGPT : 텍스트 파운데이션 모델
- 3가지 구성요소 :
- 1. 빅데이터 : 데이터가 늘어날수록 성능이 증가
- 2. 자가 학습 알고리즘 : 정답을 알려줄 필요가 없음.
- 3. 어탠션 기반 트랜스포머 모델 : 더 많은 데이터를 보다 잘 학습하고 이해할 수 있는 인공신경망 구조

# 텍스트 파운데이션 모델
- 파운데이션 모델의 3가지 구성요소 in 언어 모델
- GPT-1, BERT와 같은 언어 모델에도 3가지 구성요소 포함되있음.
- 특이점 : 
- 1. 규모의 법칙 : 더 많은 데이터, 큰 모델, 긴 학습 -> 더 좋은 성능
- 2. 창발성 : 특정 규모를 넘어서면 갑자기 모델에서 발현되는 성질
- 텍스트 파운데이션 모델 or LLM : 기존 대비 더 큰 모델이, 더 많은 데이터에서 학습되어 창발성이 나타나기 시작한 언어 모델
- 거대 언어 모델 예시 : closed 거대 언어 모델(ChatGPT, Claude, Gemini) / 개방형 거대 언어 모델(LLaMA, Gemma, Qwen)

# 거대 언어 모델의 학습
- 다양한 테스크에 대해서 동시에 학습됨.
- GPT-3 : 거대 언어 모델의 시초(학습 방법 : 다음 토큰 예측, 학습 데이터 : 3000억 토큰, 학습 비용 : 150억원 정도로 추산)
- 다음 토큰 예측 기반 거대 언어 모델의 한계 : 지시에 대해 올바르지 않은 응답을 생성하거나, 유해한 응답을 생성할 수도 있음.
- 정렬 학습 : 거대 언어 모델의 출력이 사용자의 의도와 가치를 반영하도록 하는 것
- (1) 지시 학습 : 지시에 대해 어떤 응답이 생성되어야 하는지 -> 학습 방법 자체는 BERT에서의 지도 추가학습과 동일함, 거대 언어 모델을 다양한 지시 기반 입력과 이에 대한 응답으로 지도 추가 학습(FLAN), 
- (2) 선호 학습 : 상대적으로 어떤 응답이 더 선호되어야 하는지 -> 아는 정답이 정해져 있는 객관적 테스크에서는 자연스러움. / 사람의 피드백을 통한 강화학습(RLHF),
1. 지시 학습을 통한 텍스트 파운데이션 모델 추가 학습
2. 사람의 선호 데이터를 수집하여, 보상 모델을 학습.
3. 보상이 높은 응답을 생성하도록 강의 학습을 통해 추가 학습.(사람의 개입 없이 학습된 모델들을 통해 추가 학습이 진행.)
- 선호 학습 예시(LLaMA2) : RLHF와 대화 데이터를 활용한 LLaMA2 Chat 모델 공개


# 거대 언어 모델의 평가 :
- 평가 : 구축한 시스템이 실제로 잘 동작하는지를 확인하는 단계
1. 정답이 정해진 경우 : 예측과 정답을 비교하여 일치도를 측정(정확도)
2. 정답이 정해져 있지 않은 경우 : (1) 사람에 임의의 정답을 작성, 예측과 비교 / (2) 정답과 무관하게 생성 텍스트 자체의 품질만을 측정 / (3) 생성된 텍스트의 상대적 선호를 평가(예시 : LMArena)
- LLM-as-judge : 거대 언어 모델을 통해 생성 텍스트를 평가

# 디코딩 알고리즘 
- 거대 언어 모델의 자동 회귀 생성
- p(x)로부터 X 생성하는 알고리즘(다음 단어를 선택하는 방법)
- greedy decoding : 가장 확률이 높은 단어만 선택하는 방식(최선이 아닐 수 있다.)
- beam search : k개의 후보를 동시에 고려하여 누적 확률이 가장 높은 경로를 선택(계산 비용이 많이 늘어날 수 있다.)
- sampling : 거대 언어 모델이 제공한 확률을 기준으로 랜덤하게 단어를 추출함.(생성된 응답의 품질이 낮을 수 있다.)
- Temperature(T) : 하이퍼파라미터 T를 조절해 확률 분포를 조작함. T>1이면 다양한 응답, T가 작으면 다양성 부족
- Top-K sampling : 확률이 높은 상위 K개의 토큰 중에서만 샘플링하여 잡음 단어 배제함. 문맥에 따라 불균형 발새
- Top-P sampling : 누적 확률이 특정 기준을 통과할 때를 고려함.(경우에 따라 랜덤성 발생)

# 프롬프트 엔지니어링
- 입력 프롬프트 = 지시 + 예시
- 프롬프트 엔지니어링 : 인공지능(특히 LLM)으로부터 최적의 결과물을 이끌어내기 위해 질문이나 지시사항(프롬프트)을 설계, 최적화하는 기술
- 0-shot CoT 프롬프팅 : 기존 제로샷 프롬프팅보다 훨씬 높은 추론 성능
