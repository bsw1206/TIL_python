# 원 핫 인코딩
- 규칙 기반 혹은 자연어 처리 연구의 대다수를 단어를 원자적 기호로 취급함.(0, 1로만 표현함.)
- (0,1로만 표현)단어를 원-핫 표현으로 바꾸는 과정을 원-핫 인코딩이라고 함.

# 원 핫 인코딩의 문제점
1. 차원의 저주 : 고차원의 희소 벡터를 다루기 위해선 많은 메모리 필요, 차원이 커지면 데이터가 희소해져 활용이 어렵다.
2. 의미적 정보 부족 : 비슷한 단어라도 유사한 벡터로 표현되지 않는다.

# 워드 임베딩 
- 단어를 주변 단어들로 표현?
- 단어를 단어들 사이의 의미적 관계를 포착할 수 있는 밀집되고 연속적/분산적 벡터 표현으로 나타내는 방법.
- 원 핫 인코딩과 다르게 두 단어의 벡터가 공간상 가깝게 위치함 -> 의미적 유사성 부여.
- 대표 기법 : Word2Vec(구글에서 개발)
- Word2Vec의 두 가지 알고리즘 : Skip-grams(중심 단어로 주변 단어 예측) 방식, Continuous Bag of Words(주변 단어로 중심 단어 예측) 방식
- SG vs CBOW -> SG : 적은 데이터에도 잘 동작하지만 학습 속도가 느림 / CBOW : 학습 속도가 빠르지만 희귀 단어 표현에 약하다.

# 순차적 데이터
- 데이터가 입력되는 순서와 이 순서를 통해 입력되는 데이터들 사이의 관계가 중요한 데이터
- 특징 : 
- 1. 순서가 중요하다.
- 2. 장기 의존성 : 멀리 떨어진 과거의 정보가 현재/미래에 영향을 준다.
- 3. 가변 길이 : 길이가 일정하지 않고, 단어 수도 제각각

# RNN 
- 전통적인 인공신경망(MLP, CNN)들은 고정된 길이의 입력을 받아 가변 길이의 데이터를 처리하기 쉽지 않음.
- 가변 길이의 데이터를 처리하기에 적합하다. / 이전 입력을 기억할 수 있기 때문에 순차적 데이터 처리에 적합한 구조
- 이전 시점의 정보를 담는 hidden state를 가지고 있음.
- 각 시점마다 recurrence 수식을 적용하여 hidden state를 업데이트 함.
- y=Wh, h =tanh(Wh(t-1) + Wx(t)) (encoder -> decoder로)
- 특징 : 한 번에 하나의 요소를 처리함, hidden state를 유지하면서 가변 길이 데이터를 처리할 수 있다, 과거의 입력에 영향을 받기 떄문에 feedforward 신경망과 다르다.
- 한계 : 기울기 소실의 문제(역전파 과정에서 작은 값들이 곱해짐.)

# LSTMs
- 기울기 소실 문제를 해결하기 위한 RNN의 한 종류
- cell state(장기 기억 저장)에서 정보를 읽고, 지우고, 기록할 수 있다.
- RNN과 다르게 input(얼마나 cell state에 쓸지) gate, forget(무엇을 버리고 무엇을 유지할지) gate, output(cell state중 얼마나 hidden state로 내보낼지 결정) gate를 가지고 있음.


# 언어모델
- 인간의 두뇌가 자연어를 생성하는 능력을 모방한 모델
- 한 문장의 확률은 각 단어의 조건부 확률들의 곱으로 표현할 수 있음.
- n-gram 언어모델 : 연속된 n개의 단어 묶음을 말한다.
- 다양한 n-gram이 얼마나 자주 등장하는지 통계를 수집하고 이를 통해서 다음 단어 예측
- 사용 예시 : SMT -> 한계점? 구조적 복잡성, 많은 수작업, 언어별 자원 구축 필요 -> 이후 NMT로 넘어감.

# Seq2Seq
- NMT : 인공 신경망을 이용해 기계 번역을 수행하는 방법
- 2개의 LSTM을 이용하는 아이디어(encoder, decoder)
- encoder : 입력 문장에 담긴 정보 인코딩 / decoder : 인코딩된 정보를 조건으로 하여 타겟 문장을 생성함.
- 인코더와 디코더가 하나의 통합 네트워크로 연결됨.
- teacher forcing : 정답 단어를 디코더 입력으로 강제로 넣음으로써 안정적이고 빠르게 학습을 수행
- 토큰 출력 방법 : 가장 확률이 높은 단어 선택, but 되돌리기 불가능

# Attention
- Seq2Seq의 한계 극복 : the bottlenect problem -> 벡터 하나에 모든 문장의 의미를 압축하다보니 정보 손실이 생기는 현상.
- 디코더가 단어를 생성할 때, 인코더 전체 hidden state 중 필요한 부분을 직접 참조할 수 있도록 한다.
- 효과 :
1. NMT 성능 향상 : 필요한 부분에만 디코더가 집중함.
2. bottleneck problem 해결 : 디코더가 인코더의 모든 hidden state에 직접 접근
3. vanishing gradient 문제 완화 : attention은 멀리 떨어진 단어에도 직접 연결
4. 정렬 : 단어와 단어 간의 매핑 관계를 자연스럽게 학습 -> 정렬됨.
- 디코더와 히든 상태의 관계 : 쿼리, 인코더와 히든 상태의 관계 : values

- BLEU score : 기계 번역의 출력이 사람 번역과 얼마나 비슷한지 평가하는 지표

# Self-Attention
- RNN의 한계 : 장기 의존성, 병렬화(대규모 데이터 학습에 비효율적)
- attention 매커니즘은 한 문장 내부에서 적용함.
- query, key, value가 존재
- 한계점 : 1. 순서 정보 부재, 2. 비선형성 부족, 3. 미래 참조 문제
- 한계 해결 : 1해결. positional encoding(sinusoidal position encoding, 2해결. learned absolute position embedding), Feed-Forward Network(비선형 변환 문제를 해결시켜줌.) 추가하기, 3해결. masked self-attention(모든 단어 동시 참조를 해결)

# Transformer
- 어텐션 매커니즘을 기반으로 한 신경망 구조
- 인코더 블록 : 입력 문장을 self-attention + positional encoding + FNN으로 처리
- 디코더 블록 : 이전 단어를 masked self-attention으로 처리
- 인코더-디코더 구조로 설계된 신경망 모델
- neural machine translation task에서 당시 최고 성능, 효율적인 학습, 비용 절감, NLP의 표준 아키텍쳐

# 사전학습
- 워드 임베디에서의 한계 : 많은 데이터 시간 필요, 풍부한 문맥 정보 파악 어려움.
- 언어 모델의 사전학습 : 단어의 확률분포를 모델링
- encoder 모델의 사전학습 : 양방향 문맥을 모두 활용함. masked language model을 사용하며, BERT가 존재함.
- BERT의 학습 방법 :
1. Masked LM : 입력 토큰의 일부를 무작위로 선택 -> [MASK]토큰 치환, 랜덤 토큰 치환, 그대로 두기 / 모델이 마스트된 단어에만 집중하지 않고 다양한 문맥 표현을 학습함. 
2. Next Sentence Prediction(NSP) : NSP를 통해서 문장 간 관계를 학습함.
