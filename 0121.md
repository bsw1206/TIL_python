### 함수를 사용하는 이유?
- 코드의 중복을 방지, 재사용성이 높아지고 / 코드의 가독성과 유지보수성 향상

### 매개변수와 인자
- 매개변수 : 함수를 정의할 때, 함수가 받을 값을 나타내는 변수
- 매개변수 이름 정하기 : 정의할 때 작성한 이름을 함수 내부에서 똑같이 사용해야 하며, 가독성을 위해 x, y보다는 name, age처럼 어떤 데이터인지 알 수 있는 명확한 이름을 권장함.
- 인자 : 함수를 호출할 때, 실제로 전달되는 값
- 다양한 인자의 종류
1. 위치 인자 : 함수 호출 시 인자의 위치에 따라 전달되는 인자, 함수 호출 시 반드시 값을 전달해야 함.
2. 기본 인자 : 함수 정의에서 매개변수에 기본 값을 할당하는 것(대체함), 인자를 전달하지 않으면, 기본 값이 매개 변수에 할당됨.
 ![alt text](image/image-3.png)
 -  위의 30이 기본 인자임. 
 -  greet('Bob')에서 age가 안 적히면 30으로 대체됨.
3. 키워드 인자 : 함수 호출 시 인자의 이름과 함께 값을 전달하는 인자 (ex. 함수(인자, 인자) 대신 함수(이름 = 인자, 이름 = 인자)로 쓰임)
  > **- 단 키워드 인자는 반드시 위치 인자 뒤에 있어야 함.**
  > - 이유? 순서의 모호성 때문이다. 위치 인자는 순서의 의존하여 값이 전달되는데, 키워드 인자가 순서를 무시하기 때문에 이미 순서가 깨진 상태에서 위치 인자 값을 던지면 컴퓨터가 "이 값이 몇 번째 자리에 들어가야 하는지" 판단할 수가 없음.
4. 임의의 인자 : 정해지지 않은 인자를 처리하는 인자
- 함수 정의 시 매개변수 앞에 '*'를 붙여 사용
5. 임의의 키워드 인자 목록
- 정해지지 않은 개수의 키워드 인자를 처리하는 인자
- 함수 정의 시 매개변수 앞에 '**'를 붙여서 사용
- 여러 개의 인자를 dict으로 묶음.

6. 함수 인자 권장 작성 순서
- 위치 -> 기본 -> 가변 -> 가변 키워드

### python의 범위(scope)
- global scope : 코드 어디에서든 참조할 수 있는 공간
- local scope : 함수가 만든 scope(함수 내부에서만 참조 가능)
- global variable : global scope에 정의된 변수
- local variable : local scope에 정의된 변수
- ex. ![alt text](image/image-7.png)
- 변수의 수명주기
  1. built-in scope : 파이썬이 실행된 이후로부터 영원히 유지
  2. global scope : 모듈 호출된 시점 이후 혹은 인터프리터 끝날 때까지
  3. local scope : 함수 호출 ~ 함수 종료
   - 3->2->1->으로는 들어가는데, 1->2->3->으로는 파이썬이 찾으러 들어가지 않음.
