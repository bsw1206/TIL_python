# API(Application Programming Interface)
- 두 소프트웨어(시스템)가 서로 통신할 수 있게 해주는 매커니즘
- 예시 : 소셜 로그인(특정 사이트에서 google로 계속하기), 날씨 앱(기상청(주체), 날씨 정보를 얻고 싶으면 이런 식으로 요청해야 한다는 주어진 형식이 존재함. -> 인증한 사용자들한테만 주는 행위를 취한다.)
- interface : 서로 다른 두 개의 시스템이 정보를 교환할 때, 그 사이에 존재하는 접점(예시 : 키보드, 마우스, 모니터 / TV, 리모컨)
- UI(user interface) : 사람이 소프트웨어에 접근하는 그래픽적 요소
- 클라이언트가 원하는 기능을 수행하기 위해서 서버 측에 만들어 놓은 프로그램
- API Key : API에게 요청을 보내는 애플리케이션을 구별하기 위한 고유한 식별 문자열
  - 회원가입 -> 응답( + API Key) -> 데이터 요청( + API Key) -> 응답( + 요청 데이터) 

### ex 날씨 정보?
- 찾아야 할 두 가지 : 날씨 정보를 가지고 있는 서버, 해당 서버가 제공하는 API

#### json
- JavaScript Object Notation의 약자(자바스크립트 객체 표기법)
- 경량의 텍스트 기반 데이터 형식
- 단순히 데이터를 표현(전달)하는 표현 방법 중 하나.
- 특징은 dictionary와 유사(중괄호로 둘러싸임, 키와 값 쌍의 집합으로 표현, ***key값은 문자열***, 값은 다양한 데이터 유형, 쉼표로 구분)
- 파싱(parsing) : 데이터를 의미 있는 구조로 분석하고 해석하는 과정
- json.loads() : JSON 형식의 문자열을 파싱하여 pyhton dictionary로 변환
