# ORM

- django ORM : 파이썬과 데이터베이스 사이의 통역사 역할을 수행함. 선언적 코드 만으로 데이터 저장 및 조회 명령 실행.
- 객체 지향 프로그래밍 언어의 객체와 데이터베이스의 데이터를 매핑하는 기술
- QuerySet API : ORM의 기능을 개발자가 python 코드 안에서 객체 지향적이고 직관적인 방식으로 DB를 조작할 수 있도록 제공하는 인터페이스

# QuesySet API 
- DB의 복잡한 SQL 쿼리문을 직관적인 python코드로 다룰 수 있게 해주는 강력한 번역기
- 직접 SQL을 작성할 필요 x
- 동작 방식
  1. django -> DB : 장고에서 ORM을 통해 DB로 정보를 요청할때(SQL 쿼리로 변환되어 DB로 전달됨.)
  2. DB -> django : DB가 요청에 대한 응답을 보낼 때 / 파이썬이 이해할 수 있는 python object(쿼리셋 혹은 인스턴스 형태로 변환하여 장고로 반환)
- 기본 구조 : Article.objects.all() -> Article : 모델 클래스, objects : manager, all() : Queryset API
1. article : 데이터베이스 테이블에 대한 파이썬 클래스 표현
2. .objects : 데이터베이스 조회 작업을 위한 기본 인터페이스, 모델 클래스가 DB 쿼리 작업을 수행할 수 있도록 하는 진입점
3. .all() : 특정 데이터베이스 작업을 수행하는 명령(QuerySet API 메서드), 매니저를 통해서 호출됨.

- query : 데이터베이스에 특정한 데이터를 보여 달라는 요청
- QuerySet : 데이터베이스에서 전달받은 객체 목록(데이터 모음) -> 데이터가 묶음으로 넘어오는 것
- 단 데이터베이스가 단일 객체를 반환할 때는 QuerySet이 아닌 모델(Class)의 인스턴스로 반환됨.

# 데이터 객체를 만드는 3가지 방법(save)
1. 빈 객체 생성 후 값 할당 및 DB에 저장
2. 초기 값과 함께 객체 생성 및 DB에 저장
- 1, 2 방법은 결국 .save() 메서드를 사용해야 DB에 저장됨.
- object 뒤에 1의 의미는 none에서 바뀌었으며, DB에 데이터가 저장이 된 상태
- Article.objects.all()을 사용하면 QuerySet형태로 데이터가 저장됨.
* Out[6]: <QuerySet [<Article: Article object (1)>]>
3. create()(QuerySetAPI 사) 매서드로 한 번에 생성 및 저장
- 3번은 save까지 진행시켜줌.

# 데이터 객체 불러오기(read)
- Query 반환 메서드 : all()(전체 데이터 조회), filter()(주어진 매개변수와 일치하는 객체를 포함하는 QuerySet 반환, 하나도 못 받을 때도 있고, 다 받을수도 있다.) / Query 반환하지 않는 메서드 : get(Query가 아님, 반드시 존재해야 하고, 1개만 있어야 함. 아니면 DoesNotExist or MultipleObjectReturned 등의 오류 발생)

# QuerySet API 실습
> 실습은 한 줄 씩 복사해서 붙여넣어주세요.

## Create 실습

### 첫 번째 방식
* 빈 객체 생성 후 값 할당 후 저장
```python
# 빈 객체 생성
article = Article()

# title 정보 작성 -> 인스턴스 변수(title)에 값을 할당하는 작업
article.title = 'first'

# content 정보 작성 
article.content = 'django!'

# 데이터베이스에 저장
article.save()

# 데이터베이스 Article 테이블에 저장된 모든 데이터 읽어서 확인
Article.objects.all()

```

### 두 번째 방식
```python
# 객체를 생성할 때 초기 값을 넣어서 생성
article = Article(title='second', content='django!')

# 데이터베이스에 저장
article.save()

# 데이터베이스 Article 테이블에 저장된 모든 데이터 읽어서 확인
Article.objects.all()


```

### 세 번째 방식
```python
# QuerySet API 인 create를 통해 한 번에 생성 및 저장
Article.objects.create(title='third', content='django!')

# 데이터베이스 Article 테이블에 저장된 모든 데이터 읽어서 확인
Article.objects.all()
```

## Read 실습
### all() 실습
* 모든 데이터를 조회
* 데이터가 0개 이상이 존재할 수 있음
  * QuerySet으로 결과를 반환
```python
 Article.objects.all()
```

### filter() 실습
* 주어진 매개변수와 일치하는 데이터를 반환
* 일치하는 데이터가 0개 이상일 수 있음
  * QuerySet으로 결과를 반환
```python
# content가 django!인 데이터 조회
Article.objects.filter(content='django!')

# title이 abc인 데이터 조회
Article.objects.filter(title='abc')

# title이 first인 데이터 조회
Article.objects.filter(title='first')
```
### get() 실습
* 오직 1개의 데이터를 가져올 때 사용
  * 일치하는 객체 1개만 반환
* 여러 개 있거나 없으면 에러 발생
```python
# 일치하는 데이터가 존재하는 경우
Article.objects.get(pk=1)
# 결과: <Article: Article object (1)>

# 일치하는 데이터가 없는 경우
Article.objects.get(pk=100)
# 결과: DoesNotExist: Article matching query does not exist.

# 일치하는 데이터가 여러 개 인 경우
Article.objects.get(content='django!')
# 결과: MultipleObjectsReturned: get() returned more than one Article -- it returned 2!
```

## Update 실습
1. DB에서 수정할 데이터를 가져온다.
2. 데이터를 수정한다.
3. 저장한다.

```python
# 1. 수정할 데이터를 가져온다.
article = Article.objects.get(pk=1)

# 2. 데이터를 수정한다.
article.title = 'byebye'

# 3. 저장한다.
article.save()
```

## Delete 실습

2. 삭제한다.

```python
# 1. 삭제할 데이터를 가져온다.
article = Article.objects.get(pk=1)
```
