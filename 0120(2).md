# Remote Repository
- 원격 저장소
  - 여러 개발자가 협업하고 코드를 공유할 수 있는 저장 공간
  - 온라인 상의 특정 위치에 저장
  - 다양한 원격 저장소 ***서비스*** : GitLab, GitHub, Bitbucket
- remote
  - git remote add origin remote_repo_url : 로컬 저장소에 원격 저장소 추가, origin(별칭) : 별칭을 사용해 로컬 저장소 한 개에 여러 원격 저장소를 추가할 수 있음., (remote_repo_url : 주소 추가)
- push
  - git push origin master : 원격 저장소에 commit 목록을 업로드 ("git아, push해줘 origin이라는 이름의 원격 저장소에 master라는 이름의 브랜치를"라는 의미의 명령어입니다.)
  - 명령어를 입력하면 로컬 저장소의 commit을 push
  - 핵심 : push를 진행할 때 원격 저장소에는 commit이 올라가는 것
- pull & clone
  - git pull origin master : 원격 저장소의 변경사항만을 받아옴(업데이트)
  - git clone remote_repo_url : 원격 저장소 전체를 복제(다운로드)
  - clone 받은 프로젝트는 이미 git init이 되어있음. -> 굳이 git init 명령어를 칠 필요가 없음.
  - pull을 해야 그때그때 업데이트가 되어 같은 조건에서 작업할 수 있음.
- gitignore
  - git에서 특정 파일이나 디렉토리를 추적하지 않도록 설정하는 데 사용되는 텍스트 파일(프로젝트에 따라 공유하지 않아야 하는 것들도 존재하기 때문에)
- 실습 1
  - 1. 새로운 폴더 생성 후 로컬 저장소 설정(git init)
  - 2. commit 목록 생성(add commit push)
  - 3. 새로운 GitHub repository 생성(remote)
  - 4. 원격 저장소 추가(git remote add origin master)
  - 5. commit 목록 push(push)
> 실습 2, 3도 해보기!!!

# Revert & Reset
- Git revert
  -  특정 commit을 없었던 일로 만드는 작업(그 결과를 새로운 commit으로 추가함.)
  -  git revert <commit id> : <commit id>에 해당하는 commit을 없었던 일로 되돌리기(ex. commit f9696e93ecb43133a77a8a8d66c45feb47348cfa -> commit id)
  -  기록의 무결서과 협업의 신뢰성 높임
- Git reset
  - 특정 commit으로 되돌아가는 작업
  - git reset [옵션] <commit id> : <commit id>까지 되돌아가기
  - 3가지 옵션 : --soft, --mixed, --hard / 삭제되는 commit들의 기록을 어떤 영역에 남겨둘것인지 옵션을 활용해 조정할 수 있음.
      -   --soft : 삭제된 commit들의 기록을 staging area에 남김
      -   --mixed : 삭제된 commit들의 기록을 working directory에 남김(기본값)
      -   --hard : 삭제된 commit들의 기록을 남기지 않음.

###### git restore
- modified 상태의 파일 되돌리기(수정 사항을 취소하고 원래 모습대로 되돌리는 작업.)
- 수정 취소 후에는 해당 내용을 복원할 수 없음.

#### staging area에 올라간 파일을 unstage하기
1. git restore --staged : staging area에서 working directory로 되돌리기
2. git rm --cached : git이 파악하고 있는 파일을 git에서만 제거(staging area 또는 repository에 포함된 파일을 git에서만 제거하는 것으로, 파일은 유지되어 결과적으로 add가 취소됨.)

##### 이미 삭제한 commit으로 돌아가기
- git reflog : HEAD가 이전에 가리켰던 모든 commit을 보여줌.
- Reset의 --hard 옵션을 통해 지워진 commit도 reflog로 조회하여 복구 가능
