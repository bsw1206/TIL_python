# 병합 정렬
- 정렬된 자료의 집합 병합, 한 개의 정렬된 집합으로 제작
- ex) [3]은 정렬된 걸까? -> 정렬된 것!


### log N의 의미
- N을 1이 될 때까지 반으로 나눈다.(log N번 쪼개야 함.)
- 1을 N이 될 때까지 2를 곱함.(log N이 높이)

- 병합 정렬에서는 반으로 나누는 과정에서 결국에는 N개의 데이터를 다 봐야 하므로 N logN이라는 시간 복잡도가 발생한다.


# 퀵 정렬
- 기준값을 중심으로 배열을 두 개로 분할, 각각을 정렬 -> 전체 배열 정렬
- 파티셔닝 : 기준값을 중심으로 분할하는 과정
- 시간 복잡도 : 평균은 O(NlogN), 최악은 O(N^2)
- 시간 복잡도가 다른 이유 : pivot의 설정에 따라서 달라진다(데이터의 분포에 따라서 달라진다.) 
- 데이터가 많을수록 효율적
