# 시간 복잡도
- 작업량을 표기할 때 사용됨.(연산량)
- 빅오 표기법을 사용함.
- O(3n + 2) = O(3n) = O(n)
- 3n + 2에서 최고차항만 선택, 계수 3 제거
- n개의 데이터를 입력 받아 저장한 후 각 데이터에 1씩 증가시킨 후 각 데이터를 화면에 출력하는 알고리즘의 시간 복잡도는? -> O(n)


# 배열
- 일정 자료형의 변수들을 하나의 이름으로 열거하여 사용하는 자료 구조
- 여러 개의 변수들을 효율적으로 사용 
- 다수의 변수로 하기 힘든 작업을 배열을 활용하여 쉽게 수행 가능
- 그림을 활용하여 배열을 나타내는 것이 편함.

### 1차원 배열
- 예시 : list(), [], [0] * 10

#### 입력 받은 정수를 1차원 배열에 저장하는 방법
> 6
> 
> 2 7 5 3 1 4

> N = int(input()) # N을 직접 입력, 정수로 저장
> arr = list(map(int, input().split()))

### 이론
- 배열 원소의 합 계산
- 최댓값 찾기
- 최댓값의 인덱스 찾기
- 최댓값이 여러 개인 경우 마지막 인덱스 찾기

# 정렬
- 2개 이상의 자료를 특정 기준에 의해 작은 값부터 큰 값 혹은 그 반대의 순서대로 재배열하는 알고리즘
- 버블 정렬 : 인접한 두 개의 원소를 비교하며 자리를 계속 교환하는 방법

# 알고리즘 학습 가이드
- 개념 정리를 읽어보고 문제 풀이
- IM에서 나오는 문제 풀이
- 코드 리뷰를 매일 진행(행별로)
- 코드 리뷰에서 리뷰 포인트 미리 생각하는 것이 중요함!
- 코드 리뷰 후 자기 점검 및 피드백, 다음 과제 부여
- 숙제도 가끔 출제될 수 있음
- 하루에 푸는 문제 수와 소요 시간을 선정
- 1시간 30분 ~ 2시간 이내로 한 문제 풀어보되, 난이도에 따라 조정 가능
- 매일 과제 문제와 추천 SWEA 탐색
- 쉬운 것부터 차근차근 올라가기
- 문제 분석(10~20분), 설계하기(30분 이내), 구현하기(60~120분 이내), 디버깅 및 테스트 진행(바로 코드를 작성하는 것은 좋지 않다.)


### 풀이 시 대처
- 첫 풀이는 온전히 내 힘으로!
- 시간 초과 시 어디서 막히는지 점검 시작
- 다른 사람의 제출 코드 및 해설 코드 분석
- 참고 자료를 분석한 것을 바탕으로 재시도 (20~30분)
- 다시 못 풀었다면 참고자료 돌아가기(이 과정을 2~4번 반복)
- 100% 내 힘으로 푼 문제가 아니면 기록 해두고 일주일내에 다시 풀이
- 다시 풀었을 때 어떤 실수가 줄었는지 비교


### 정리 & 복습
- 어떤 자료구조와 알고리즘을 사용했는지 메모
- 배운 점 및 실수한 부분
- **추후 복습 필요한 문제인지 여부 표시**
- 체크리스트를 통해서 틀린 문제나 못 푼 문제를 내것으로 만드는 과정이 중요함.
